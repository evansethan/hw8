The LLVM Compiler is an excellent example of a C++ program which adheres to the SOLID principles. 

S (Single Responsibility):
LLVM is made up of a long chain of "passes", which each perform one change or optimization per pass. Each pass has a single responsibility, preventing "spaghetti code" and ensuring a bug in a single pass doesn't break or affect other unrelated parts of the compiler.


O (Open/Closed):
LLVM allows developers to add custom optimization passes to the PassManager without modifying the core engine source code. Because LLVM uses modular architecture, developers can add/remove custom pluggable components without breaking anything.


L (Liskov Substitution):
The Value class is a universal base for a number of different subtypes. LLVM uses a custom Run-Time Type Information system to make sure that down casting to a subtype is consistent with the base.


I (Interface Segregation):
There isn't one enormous interface for IR objects. LLVM uses specialized classes, like GlobalValue and Instruction, which divide up functionality and prevent complex methods and nullified fields on objects that don't need them.


D (Dependency Inversion):
One could argue that the whole reason LLVM exists is to solve the low-level hardware dependency problem. It does so by using abstract interfaces like TargetMachine instead of a concrete reference. This inversion enables the IR to compile across a wide range of architectures, including x86 and ARM.
